## Local asset paths

The poster and audio paths you provided are local Windows paths that cannot be deployed directly to Vercel:

- Poster: `C:\Users\vishn\OneDrive\Pictures\WhatsApp Image 2026-06-20 at 10.16.18 AM.jpeg`
- Audio: `C:\Users\vishn\OneDrive\Pictures\WhatsApp Audio 2026-06-20 at 4.50.13 PM.mpeg`

### What to do instead

1. Create a `public` folder in `a:\cons`.
2. Copy the poster and audio files into that folder.
3. Rename them to shorter, deployment-friendly names, for example:
   - `public/poster.jpeg`
   - `public/audio.mpeg`
4. Update `Campus_constellation.html` to use relative or root-relative URLs rather than Windows paths. Example:

```html
<img src="/poster.jpeg" alt="Campus poster" />
<audio controls src="/audio.mpeg"></audio>
```

> Vercel serves files from the project root for static sites. Absolute local Windows paths will not work after deployment.

## Deploying to Vercel

1. If you have a Git repository, commit the project files and this `README.md`.
2. Push to GitHub, GitLab, or Bitbucket.
3. Connect your repository to Vercel.
4. In Vercel, select the `cons` folder as the project root if your repo contains multiple folders.
5. Set the framework preset to `Other` or `Static Site`.
6. Deploy.

## Notes for Vercel

- `Campus_constellation.html` is a static HTML file. No build step is required.
- Ensure your assets are in the project and referenced with relative paths.
- If the page still uses `C:\...` paths, Vercel will not be able to load them.
